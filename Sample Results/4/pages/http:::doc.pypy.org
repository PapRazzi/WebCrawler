<h1>Welcome to PyPy Development<a class="headerlink" href="#welcome-to-pypy-development" title="Permalink to this headline">¶</a></h1>
<p>The PyPy project aims at producing a flexible and fast <a class="reference external" href="http://docs.python.org/index.html">Python</a>
implementation.  The guiding idea is to translate a Python-level
description of the Python language itself to lower level languages.
Rumors have it that the secret goal is being faster-than-C which is
nonsense, isn&#8217;t it?  <a class="reference external" href="architecture.html#mission-statement">more...</a></p>
</div>

<div class="section" id="meeting-pypy-developers">
<h1>Meeting PyPy developers<a class="headerlink" href="#meeting-pypy-developers" title="Permalink to this headline">¶</a></h1>
<p>The PyPy developers are organizing sprints and presenting results at
conferences all year round. They will be happy to meet in person with

<div class="section" id="project-documentation">
<h1>Project Documentation<a class="headerlink" href="#project-documentation" title="Permalink to this headline">¶</a></h1>
<p>PyPy was funded by the EU for several years. See the <a class="reference external" href="http://pypy.org">web site of the EU
project</a> for more details.</p>
<p><a class="reference external" href="architecture.html">architecture</a> gives a complete view of PyPy&#8217;s basic design.</p>
<p><a class="reference external" href="coding-guide.html">coding guide</a> helps you to write code for PyPy (especially also describes
coding in RPython a bit).</p>
<p><a class="reference external" href="sprint-reports.html">sprint reports</a> lists reports written at most of our sprints, from
2003 to the present.</p>
<p><a class="reference external" href="extradoc.html">papers, talks and related projects</a> lists presentations
and related projects as well as our published papers.</p>
<p><a class="reference external" href="video-index.html">PyPy video documentation</a> is a page linking to the videos (e.g. of talks and
introductions) that are available.</p>
<p><a class="reference external" href="index-report.html">Technical reports</a> is a page that contains links to the
reports that we submitted to the European Union.</p>
<p><a class="reference external" href="dev_method.html">development methodology</a> describes our sprint-driven approach.</p>
<p><a class="reference external" href="https://bitbucket.org/pypy/pypy/src/default/LICENSE">LICENSE</a> contains licensing details (basically a straight MIT-license).</p>
<p><a class="reference external" href="glossary.html">Glossary</a> of PyPy words to help you align your inner self with
the PyPy universe.</p>
</div>
<div class="section" id="status">
<h1>Status<a class="headerlink" href="#status" title="Permalink to this headline">¶</a></h1>
<p>PyPy can be used to run Python programs on Linux, OS/X,
Windows, on top of .NET, and on top of Java.
To dig into PyPy it is recommended to try out the current
Mercurial default branch, which is always working or mostly working,
instead of the latest release, which is <a class="reference external" href="release-2.0.0-beta1.html">2.0 beta1</a>.</p>
<p>PyPy is mainly developed on Linux and Mac OS X.  Windows is supported,
but platform-specific bugs tend to take longer before we notice and fix
them.  Linux 64-bit machines are supported (though it may also take some
time before we notice and fix bugs).</p>
<p>PyPy&#8217;s own tests <a class="reference external" href="http://buildbot.pypy.org/summary">summary</a>, daily updated, run through BuildBot infrastructure.
You can also find CPython&#8217;s compliance tests run with compiled <tt class="docutils literal"><span class="pre">pypy-c</span></tt>
executables there.</p>
</div>
<div class="section" id="source-code-documentation">
<h1>Source Code Documentation<a class="headerlink" href="#source-code-documentation" title="Permalink to this headline">¶</a></h1>
<p><a class="reference external" href="objspace.html">object spaces</a> discusses the object space interface
and several implementations.</p>
<p><a class="reference external" href="interpreter.html">bytecode interpreter</a> explains the basic mechanisms
of the bytecode interpreter and virtual machine.</p>
<p><a class="reference external" href="interpreter-optimizations.html">interpreter optimizations</a> describes our various strategies for
improving the performance of our interpreter, including alternative
object implementations (for strings, dictionaries and lists) in the
standard object space.</p>
<p><a class="reference external" href="translation.html">translation</a> is a detailed overview of our translation process.  The
<a href="#id25"><span class="problematic" id="id26">rtyper_</span></a> is the largest component of our translation process.</p>
<p><a class="reference external" href="https://bitbucket.org/pypy/extradoc/raw/tip/eu-report/D05.1_Publish_on_translating_a_very-high-level_description.pdf">dynamic-language translation</a> is a paper that describes
the translation process, especially the flow object space
and the annotator in detail. (This document is one
of the <a class="reference external" href="index-report.html">EU reports</a>.)</p>
<p><a class="reference external" href="low-level-encapsulation.html">low-level encapsulation</a> describes how our approach hides
away a lot of low level details. This document is also part
of the <a class="reference external" href="index-report.html">EU reports</a>.</p>
<p><a class="reference external" href="translation-aspects.html">translation aspects</a> describes how we weave different
properties into our interpreter during the translation
process. This document is also part of the <a class="reference external" href="index-report.html">EU reports</a>.</p>
<p><a href="#id27"><span class="problematic" id="id28">`garbage collector`_</span></a> strategies that can be used by the virtual
machines produced by the translation process.</p>
<p><a class="reference external" href="parser.html">parser</a> contains (outdated, unfinished) documentation about
the parser.</p>
<p><a class="reference external" href="rlib.html">rlib</a> describes some modules that can be used when implementing programs in
RPython.</p>
<p><a class="reference external" href="config/">configuration documentation</a> describes the various configuration options that
allow you to customize PyPy.</p>
<p><a href="#id29"><span class="problematic" id="id30">`CLI backend`_</span></a> describes the details of the .NET backend.</p>
<p><a href="#id31"><span class="problematic" id="id32">`JIT Generation in PyPy`_</span></a> describes how we produce the Python Just-in-time Compiler
from our Python interpreter.</p>
